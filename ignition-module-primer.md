# Ignition Module Builder Primer

A working reference for building Ignition modules from scratch using the SDK. Covers 8.1 patterns with 8.3 migration notes.

---

## 1. What Is a Module?

A module is a `.modl` file — a signed ZIP archive containing:
- One or more compiled JAR files (scoped to Gateway, Designer, Client)
- A `module.xml` manifest
- Optional license and documentation HTML files

The platform loads the module, reads the manifest, locates the hook classes, and starts them up. Nearly everything in Ignition's commercial stack (Vision, SQLBridge, OPC-UA, Perspective) is itself a module built with this same SDK.

| Requirement | Notes |

```

```

## 4. Project Structure

A typical multi-scope Gradle module project looks like:

```
my-module/
├── build.gradle.kts          # Root build — applies the IA Gradle plugin
├── settings.gradle.kts       # Declares subprojects
├── gradle.properties         # Module signing credentials, etc.
├── common/                   # Scope: G + C + D (shared interfaces, DTOs)
│   └── src/main/java/...
├── gateway/                  # Scope: G
│   └── src/main/java/.../GatewayHook.java
├── designer/                 # Scope: D
│   └── src/main/java/.../DesignerHook.java
└── client/                   # Scope: C (Vision clients)
    └── src/main/java/.../ClientHook.java
```

Not all scopes are required. A pure gateway module only needs the `gateway` subproject.

## 5. The Gradle Plugin

The IA Gradle plugin handles the `.modl` assembly — it generates `module.xml`, packages JARs by scope, signs the archive, and publishes.

**Root `build.gradle.kts`:**
```kotlin
plugins {
    id("io.ia.sdk.modl") version "0.3.0" // check latest on Gradle Plugin Portal
}

ignitionModule {
    name.set("My Module")
    fileName.set("my-module")
    id.set("com.example.my-module")
    moduleVersion.set("1.0.0")
    requiredIgnitionVersion.set("8.1.11")

    // Map of gradle subproject path -> scope string
    projectScopes.putAll(mapOf(
        ":gateway" to "G",
        ":common"  to "GCD",
        ":designer" to "D",
        ":client"  to "C"
    ))

    // Map of module ID -> scope for module dependencies
    moduleDependencies.set(mapOf(
        "com.inductiveautomation.opcua" to "G"
    ))

    // Map of fully qualified hook class -> scope
    hooks.putAll(mapOf(
        "com.example.mymodule.gateway.GatewayHook" to "G",
        "com.example.mymodule.designer.DesignerHook" to "D",
        "com.example.mymodule.client.ClientHook" to "C"
    ))

    // Set true in dev; must be false for production signing
    skipModlSigning.set(false)
}
```

**Build the `.modl`:**
```bash
./gradlew build
# Output: build/my-module-1.0.0.modl
```

**Developer mode upload** (skip manual install):
Add to `ignition.conf`:
```
wrapper.java.additional.[N]=-Dia.developer.moduleupload=true
```
Then run:
```bash
./gradlew deployModl  # uploads directly to dev gateway
```

## 6. The `module.xml` Manifest

Auto-generated by the Gradle plugin, but worth understanding:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<modules>
  <module>
    <id>com.example.my-module</id>
    <name>My Module</name>
    <description>Does something useful.</description>
    <version>1.0.0</version>
    <requiredignitionversion>8.1.11</requiredignitionversion>
    <requiredframeworkversion>8</requiredframeworkversion>
    <jar scope="G">my-module-gateway-1.0.0.jar</jar>
    <jar scope="GCD">my-module-common-1.0.0.jar</jar>
    <jar scope="D">my-module-designer-1.0.0.jar</jar>
    <hook scope="G">com.example.mymodule.gateway.GatewayHook</hook>
    <hook scope="D">com.example.mymodule.designer.DesignerHook</hook>
    <depends>com.inductiveautomation.opcua</depends>
  </module>
</modules>
```

**Scope codes:**
- `G` = Gateway
- `D` = Designer
- `C` = Vision Client (not Perspective)
- Combined: `GCD`, `GC`, etc.

## 7. Scopes, Hooks, and Contexts

| Scope | Hook Interface | Abstract Base | Context Passed In |
|---|---|---|---|
| Gateway | `GatewayModuleHook` | `AbstractGatewayModuleHook` | `GatewayContext` |
| Designer | `DesignerModuleHook` | `AbstractDesignerModuleHook` | `DesignerContext` |
| Vision Client | `ClientModuleHook` | `AbstractClientModuleHook` | `ClientContext` |

Always extend the `Abstract*` base — it provides empty implementations so you only override what you need.

## 8. The Gateway Hook Lifecycle

```java
public class GatewayHook extends AbstractGatewayModuleHook {

    private GatewayContext context;

    /**
     * setup() — called BEFORE startup.
     * - Register extension point types
     * - Update PersistentRecord schemas
     * - Store the GatewayContext — you'll need it everywhere
     * - DO NOT modify internal DB here; DO NOT rely on other modules being loaded
     */
    @Override
    public void setup(GatewayContext context) {
        this.context = context;
        // Register schema updates here:
        // context.getSchemaUpdater().updatePersistentRecords(MyRecord.META);
    }

    /**
     * startup() — system is nearly fully running.
     * - Start services, register listeners, initialize business logic
     * - LicenseState lets you gate features on licensing
     */
    @Override
    public void startup(LicenseState licenseState) {
        // Example: register a scripting module
        // Example: start a background thread / ExecutionManager task
    }

    /**
     * shutdown() — module is being removed or gateway is stopping.
     * CRITICAL: unregister EVERYTHING you registered. Memory leaks here
     * are especially painful when modules are reloaded multiple times.
     */
    @Override
    public void shutdown() {
        // Unregister services, stop threads, release resources
    }
}
```

## 9. GatewayContext — The Master Key

`GatewayContext` is passed to `setup()` and should be stored as a field. It's your gateway to every platform subsystem. Valid for the entire gateway lifetime — safe to use from any method.

### Key accessors:

| Method | What you get |
|---|---|
| `getTagManager()` | `GatewayTagManager` — read/write tags, create providers |
| `getDatasourceManager()` | DB connections by name |
| `getPersistenceInterface()` | Internal DB (w/ redundancy replication) |
| `getLocalPersistenceInterface()` | Internal DB (direct, bypasses redundancy) |
| `getAlarmManager()` | Query/subscribe to alarm events |
| `getProjectManager()` | Access and manage projects/resources |
| `getExecutionManager()` | Shared scheduled executor for background tasks |
| `getScheduledExecutorService()` | Raw `ScheduledExecutorService` |
| `getModuleServicesManager()` | Register/consume `ModuleService` instances |
| `getUserSourceManager()` | Configured identity providers |
| `getAuditManager()` | Audit log |
| `getStoreAndForwardManager()` | S&F pipeline for history |
| `getTagHistoryManager()` | Tag history queries |
| `getGatewayNetworkManager()` | GAN — inter-gateway communication |
| `getMessageDispatchManager()` | Push messages to clients/designers |
| `getLicenseManager()` | License details |
| `getSchemaUpdater()` | Register PersistentRecord schemas |
| `getSecurityZoneManager()` | Security zones |
| `getOpcManager()` | OPC connections |
| `getWebResourceManager()` | Mount static web resources |
| `getMetricRegistry()` | Dropwizard metrics |
| `getLoggingManager()` | Gateway logging subsystem |
| `getRedundancyManager()` | Redundancy status/events |
| `getEventBus()` | Platform-wide event bus (from `CommonContext`) |
| `getScriptManager()` | Script execution (from `CommonContext`) |

## 10. Adding Scripting Functions

Register a `system.yournamespace.*` function visible in all scopes.

**Step 1 — Define interface in `common` scope:**
```java
// common/src/.../MyScriptFunctions.java
public interface MyScriptFunctions {
    public String doSomething(String input);
}
```

**Step 2 — Implement on the Gateway:**
```java
// gateway/src/.../GatewayScriptModule.java
public class GatewayScriptModule extends AbstractScriptModule
        implements MyScriptFunctions {

    @Override
    protected Object execute(String functionName, Object[] args) {
        // Called when executed in Gateway scope directly
        return null;
    }

    @Override
    public String doSomething(String input) {
        return "Hello from gateway: " + input;
    }
}
```

**Step 3 — Register in GatewayHook:**
```java
@Override
public void initializeScriptManager(ScriptManager manager) {
    super.initializeScriptManager(manager);
    manager.addScriptModule(
        "system.mymodule",
        new GatewayScriptModule(),
        new PropertiesFileDocProvider()
    );
}
```

**Step 4 — Client/Designer RPC delegation:**
```java
// client/src/.../ClientScriptModule.java
public class ClientScriptModule extends AbstractScriptModule
        implements MyScriptFunctions {

    private final MyScriptFunctions rpc;

    public ClientScriptModule() {
        // Proxies calls over the wire to GatewayHook.getRPCHandler()
        rpc = ModuleRPCFactory.create(
            "com.example.my-module",   // must match module ID
            MyScriptFunctions.class
        );
    }

    @Override
    public String doSomething(String input) {
        return rpc.doSomething(input);  // delegates to gateway
    }
}
```

**Step 5 — Return RPC handler from GatewayHook:**
```java
@Override
public Object getRPCHandler(ClientReqSession session, String projectName) {
    return new GatewayScriptModule();
}
```

## 11. Persistent Records (Internal DB / Config Storage)

`PersistentRecord` maps a Java class to a table in Ignition's internal H2 database. Used for storing module configuration (like how drivers, connections, and notification profiles store their settings).

```java
public class MyConfigRecord extends PersistentRecord {

    public static final RecordMeta<MyConfigRecord> META =
        new RecordMeta<>(MyConfigRecord.class, "MyConfig");

    public static final StringField Name =
        new StringField(META, "Name").setDefault("default");
    public static final LongField Interval =
        new LongField(META, "Interval").setDefault(5000L);
    public static final BooleanField Enabled =
        new BooleanField(META, "Enabled").setDefault(true);

    @Override
    public RecordMeta<?> getMeta() { return META; }
}
```

**Register the schema in `setup()`:**
```java
@Override
public void setup(GatewayContext context) {
    this.context = context;
    try {
        context.getSchemaUpdater().updatePersistentRecords(MyConfigRecord.META);
    } catch (Exception e) {
        logger.error("Error updating schema", e);
    }
}
```

**Read/write records:**
```java
// Query all records
PersistenceInterface db = context.getPersistenceInterface();
List<MyConfigRecord> records = db.query(
    QueryBuilder.select(MyConfigRecord.META).build()
);

// Create a new record
MyConfigRecord rec = db.createNew(MyConfigRecord.META);
rec.setString(MyConfigRecord.Name, "my-config");
db.save(rec);

// Delete
db.delete(MyConfigRecord.META, recordId);
```

## 12. Gateway Config Pages

To add a config page under `Configure > Your Module` in the Gateway web UI, override in `GatewayHook`:

```java
@Override
public List<? extends IConfigTab> getConfigPanels() {
    return List.of(
        DefaultConfigTab.tabBuilder(MyConfigPage.class)
            .name("mymodule")
            .title("My Module Config")
            .build()
    );
}

@Override
public List<ConfigCategory> getConfigCategories() {
    return List.of(
        new ConfigCategory("mymodule", "My Module", 700)
    );
}
```

Config pages use Wicket components — extend `ConfigPanel`.

## 13. Module Services (Cross-Module Communication)

If you want to expose an API to *other* modules:

```java
// Define service interface (in common, or your gateway jar)
public interface MyModuleService extends ModuleService {
    String doThing();
}

// Implement and register in startup():
context.getModuleServicesManager().registerService(
    MyModuleService.class,
    new MyModuleServiceImpl()
);

// Consuming module looks it up:
Optional<MyModuleService> svc = context
    .getModuleServicesManager()
    .getService(MyModuleService.class);
```

## 16. 8.3 Migration Notes

If targeting 8.3, be aware of these breaking changes:

| 8.1 Pattern | 8.3 Replacement |
|---|---|
| `getRPCHandler()` / Java serialization RPC | `@RpcInterface` annotation + `GatewayRpcImplementation.of()` + `GatewayConnection.getRpcInterface()` |
| `PersistentRecord` for config storage | New **Config Resource API** (declarative resource-based) |
| `EncodedStringField` for secrets | `SecretConfig` field type via `DefaultRecordEncodingDelegate` |
| `Resource.getData()` returning `byte[]` | Returns `Optional<ImmutableBytes>` |
| `ProjectManifest.getTitle()` | `ResourceCollectionManifest.title()` |

Full detail: https://www.sdk-docs.inductiveautomation.com/docs/8.3/to-83-upgrade-guide/

## 17. Installing & Testing

1. Build: `./gradlew build` → generates `.modl` in `build/`
2. Gateway web UI → **Config > Modules** → scroll to bottom → **Install or Upgrade a Module**
3. Select your `.modl` file and install
4. Check Gateway logs for hook lifecycle messages
5. For faster iteration: use `deployModl` Gradle task (requires `Dia.developer.moduleupload=true`)
